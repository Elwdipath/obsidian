# Single-function problems

Often called a “toy problem,” a single-function problem is one that can be solved by writing a single function.

### Example

> Given a sorted list of integers, square the elements and give the output in sorted order.
For example, given `[-9, -2, 0, 2, 3]`, return `[0, 4, 4, 9, 81]`.
> 

## Leetcode grinding

The most common source of these problems is [Leetcode](https://leetcode.com/problemset/all/). Thus we have the term “Leetcode grinding” — i.e., solving many Leetcode problems to learn the patterns and “game” the interview.

Frankly speaking, this problem type is widely reviled as being unrelated to actual work. Some problems are especially galling as they rely on some “trick” to reach a solution, and are thus, practically speaking, rarely solvable from first principles, under normal time constraints.

## So, why are they even asked?

Charitably speaking, it’s because these problems are deemed to do an OK job,  in a relatively short time, eliciting:

- requirements verification
- face-to-face technical communication
- systematic planning and implementation
- the ability to write functioning code in some language
- some small-scale clean coding
- some computer science fundamentals, focused on data structures & algorithms
- basic testing

Obviously, the performance that a single-function problem can elicit in just 45 minutes is hardly sufficient. But as one set of signals in a larger process, it’s not *terrible* in itself, if the team deploying such questions is doing so in an enlightened, friendly way.

## How to prep for single-function problems

First, [Define your recipe for live problem-solving](Define%20your%20recipe%20for%20live%20problem-solving%20dc411ab064a18304bc40010d65038681.md).

Then, consider that you need to review your CS fundamentals.

That covers a lot of topics, so the most efficient route is to make use of a good prep book or short prep course.

Here are our staff picks:

- [book] [Elements of Programming Interviews](https://www.amazon.com/Elements-Programming-Interviews-Python-Insiders/dp/1537713949/) (this is in Python; there’s also a Java version)
- [course] [Algo Monster](https://algo.monster/)
- [course] [Educative: Grokking the Coding Interview](https://www.educative.io/courses/grokking-the-coding-interview?)

Leetcode itself has some preparatory materials too, and there are in fact many more resources than just the above. We’re just presenting a curated shortlist.

These are excellent standalone resources. But along with them, if you would like some lightweight, personalized help, just join an Underdog cohort and reach out to the staff by emailing community@underdog.io.
