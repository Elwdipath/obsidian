# System design problems

<aside>

</aside>

“System design” interview problems are usually focused on architecture of large-scale, distributed web systems.

Common examples of these problems would be:

- Design a vending machine
- Design a URL shortener
- Design Dropbox
- Design YouTube

On the face of it, something like “Design YouTube” seems absurd to tackle in a one-hour slot. Of course the interviewers aren’t expecting a complete, actionable design in so little time. At a very high level, they’re looking for:

- a structured problem-solving approach
- considering tradeoffs
- familiarity with typical problem patterns that arise in larger systems
- familiarity with best-practice resolutions to those problems
- familiarity with common tools that implement those best practices

… along with the usual soft skills of clear, collaborative technical communication and so on.

## Who does NOT need to prep for system design problems

Before we even talk about prep resources, perhaps you don’t need to deal with this type of problem at all.

Generally, front-end specialists, machine learning engineers and so forth, aren’t often asked this sort of problem. It’s more for full-stack or back-end engineers.

So ignore this problem type, if it’s not relevant to you.

## How to prep for system design problems

There are some excellent books and short courses written about this subject:

- [course] [Educative: Grokking the System Design Interview](https://www.educative.io/courses/grokking-modern-system-design-interview-for-engineers-managers)
- [book] [System Design Interview: Insider’s Guide](https://www.amazon.com/System-Design-Interview-insiders-Second/dp/B08CMF2CQF)
- [book] [System Design Interview: Insider’s Guide, Vol. 2](https://www.amazon.com/System-Design-Interview-Insiders-Guide/dp/1736049119)

These are excellent standalone resources.
